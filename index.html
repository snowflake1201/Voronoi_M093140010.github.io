演算法設計與分析

題目：

撰寫課本的Voronoi diagram 演算法。

資安碩一

王品媛

M093140010

軟體說明

壓縮檔裡有"main.exe"檔案，在window10的作業系統下直接雙擊執行就可以啟動(若視窗沒有馬上跳出需耐心等待)。

以下分六個按鈕功能做介紹：

Run

在畫布點選完或讀取完檔案後點擊，將完整執行Voronoi diagram。

Step by Step

在畫布點選完或讀取完檔案後持續點擊，將分步驟執行Voronoi diagram。

Read File

可選取輸入檔案，一次只能讀一筆。

Read Output File

目前無法使用。

Output Result

輸出執行完的結果。

Clean All

清空畫布。

軟體規格書

執行軟體的方式和輸入輸出資料的規格。

輸入資料

必須使用老師的範例檔案的格式，一次一筆資料。

輸出資料

只輸出座標的陣列。

軟體介面

畫布大小為600x600，可自由點擊.

重複測試

每次測試完需按"Clean All"按鈕清理陣列，或直接重啟程式。




程式設計

一開始就把三個點做divide and conquer分為1點和2點，再做merge，超過3點以上就不斷遞迴直到小於三點，2點取中垂線，1點直接輸出。




軟體測試與實驗結果

不同輸入值的實驗結果。

2個點

求出中垂線，沒有任何問題。

3個點

求出Convex Hull，1點和2點merge起來。

4個點

切成2點和2點，因為沒有消線，畫面開始混亂。

5個點

遞迴開始，未消線導致更加混亂。

6個點

同5個點。

10個點

線條打架。

12個點

未消掉的線嚴重干擾merge了。

結論與心得

這次寫voronoi的過程真的非常艱辛，一度懷疑自己寫不出來，初測時連線都畫不好，後來經過學長和老師的鼓勵，不斷地請教同學、更改程式思路，從最底層就開始merge和遞迴，才終於實作出來，可惜消線的部分還是沒有想到好方法，導致多點之後畫面變得超級混亂，分割線一直撞到不應該存在的線，還有很多特殊條件沒有另外處理，所以遇到多點同線和正多邊形會爆錯，總體來說學到非常多，謝謝老師和各位同學、助教。
